win上卸载mysql

Posted on October 24, 2018

windows上卸载mysql

很多情况我们发现mysql数据库服务启动不了。或者忘记了数据库密码。导致不能登录，需要重新按照这时候就首先需要卸载干净再去装。这里我们重点说下如何在win上完全卸载mysql
1.停掉服务 cmd里面执行net stop mysql

2.删除服务 cmd里面执行sc delete mysql

3.卸载mysqlserver软件

4.删除安装的文件夹 一般默认安装的路径为

C:\ProgramData\MySQL
c:\Programx86\MySQL

5.删除注册表

  • 删除HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\Eventlog\Application\MySQL文件夹
  • 删除HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Eventlog\Application\MySQL的文件夹。

6.机器重启下。即可完全卸载

yield与生成器

Posted on August 28, 2018

python 生成器（yield）

生成器

为了理解yield,就首先要明白什么是生成器。可是要理解生成器需要知道什么是可迭代对象。因为yield就是生成器每次只会返回一个内容。

可迭代对象

当我们创建一个字符串后可以逐项去读取每个字母。这叫做一个可迭代对象。

for c in 'hello':
    print c

再比如我们用列表推到式创建的列表也可以叫做可迭代对象

genter=[x*x for x in range(3)]
for i in genter:
    print (i)

生成器

生成器是可迭代的，我们只能读取它一次，因为它并不把所有的值放到内存中，它是实时生成数据。例如

genter=(x*x for x in range(3))
for i in genter:
    print (i)
看起来似乎和上面的列表没有什么区别。但是确是一个生成器对象
上面的可迭代器对象的列表可以反复调用for.但是我们的生成器只能迭代一次有效。即运行多次for只会有1次出现第一次的结果。

yield

yield是个关键字。和函数里面的return差不多。只是返回的是一个生成器。看下下面的code:

 def createGenerator():
     mylist=range(3)
     for i in mylist:
        yield i*i
gen=createGenerator()
print (gen) #这里返回一个生成器对象<generator object createGenerator at 0x02C464B8>

上面的例子并没有什么是实际的效果。目的就是只是知道系统会返回一堆只需要读取一次的内容。当调用函数的时候函数里面的代码并不是立马执行的。这个函数只是返回一个生成器对象。函数里面的代码只有当使用for循环的时候才会执行。

第一次迭代中你的函数会执行，从开始到达 yield 关键字，然后返回 yield 后的值作为第一次迭代的返回值. 然后，每次执行这个函数都会继续执行你在函数内部定义的那个循环的下一次，再返回那个值，直到没有可以返回的。如果生成器内部没有定义 yield 关键字，那么这个生成器被认为成空的。这种情况可能因为是循环进行没了，或者是没有满足 if/else 条件。

yield demo Example

 class Back(object):
     crisis=False
     def create_atm(self):
        while not self.crisis:
            yield '$100'
hsb=Back()
ns=hsb.create_atm()
print (ns)
print (ns.next())
print (ns.next())
print ([ns.next() for i in range(5)])
hsb.crisis=True
#print(ns.next())  # stopIteration 不满足条件
ls=hsb.create_atm()   
#print (ls.next())   # StopIteration
# print '*'*20
hsb.crisis=False #清空
print (ns.next())
bus=hsb.create_atm()
for cash in bus:
   print cash       #条件为真的一直执行下去

python模块之glob

Posted on August 24, 2018

python之glob模块那点事儿

glob的背景

glob的匹配符

glob查找文件只用到3个匹配符号 * ？ [] 
* 代表 0个或者多个字符
？代表1个字符
[] 代表的是查找的范围 如[0-9]匹配数字

glob模块的方法

1.glob.glob()

该方法返回所有匹配的文件路径列表。该方法需要一个参数用来指定需要匹配的路径字符串（字符串可以为绝对路径也可以为相对路径）其中返回的文件路径列表只会返回当前路径中匹配到的。还是先上一段代码：

import glob
file=glob.glob(r'*.py') #列出当前路径所匹配的所有的.py的文件 
file1=glob.glob(r'C:\Users\admin\Pictures\Saved Pictures\*.jpg')  # 这里是去绝对路径里面去寻找
## 上面的返回都是一个list

2.glob.iglob()

获取一个迭代器（iterator ）对象，使用它可以逐个获取匹配的文件路径名。与glob.glob()的区别是一次只获取一个文件路径名。而glob.glob()一次获取所有的匹配的文件名。接着上代码

  import glob
  filelist=glob.iglob(r'*.py')
  print filelist # <generator object iglob at 0x02DA9FD0>
  for file in filelist:
      print file       #一次只返回一个

redis数据结构

Posted on July 24, 2018

redis

Redis 是完全开源免费的，遵守BSD协议，是一个高性能的key-value数据库。
Redis 与其他 key - value 缓存产品有以下三个特点：
1.Redis支持数据的持久化，可以将内存中的数据保存在磁盘中，重启的时候可以再次加载进行使用。
2.Redis不仅仅支持简单的key-value类型的数据，同时还提供list，set，zset，hash等数据结构的存储。

  1. 列表

列表详常见的操作：
可以用空格分开一次插入多个值
lpush key value 向列表的前面插入（lpushx 当健存在时才插入）
rpush key value 向列表的后面插入 (rpushx 当健存在时才插入)
lindex key index 根据索引获取列表的健
lrange key start stop 获取从索引start开始到stop为止的范围内的内容

集合

向集合中添加元素
sadd movie the first world 向集合中一次添加三个元素。
scard movie 获取set中的成员个数
smembers movie 获取set中的元素。
spop movie 2 随机从集合中弹出两个元素。
sismember age movie 判断是否是set的成员。

有序集合

zadd sql 1 redis 向有序集合添加元素。这里需要给个分数
zcard sql 获取有序集合的成员数
zconut sql 1 2 获取分数在1-2之间的成员个数
zrank sql memcache 获取成员的索引，如果没有返回 nil
zrange sql 0 10 withscores 进行打分。

hyperloglog

1.Redis HyperLogLog 是用来做基数统计的算法，HyperLogLog 的优点是，在输入元素的数量或者体积非常非常大时，计算基数所需的空间总是固定 的、并且是很小的。
2.在 Redis 里面，每个 HyperLogLog 键只需要花费 12 KB 内存，就可以计算接近 2^64 个不同元素的基 数。这和计算基数时，元素越多耗费内存就越多的集合形成鲜明对比。
但是，因为 HyperLogLog 只会根据输入元素来计算基数，而不会储存输入元素本身，所以 HyperLogLog 不能像集合那样，返回输入的各个元素.

什么是基数呢？

比如数据集 {1, 3, 5, 7, 5, 7, 8}， 那么这个数据集的基数集为 {1, 3, 5 ,7, 8}, 基数(不重复元素)为5。
pfadd log “error” 添加一个到数据集
pfcount log 获取统计log健的基数
PFMERGE destkey sourcekey 将两个hyperlog合并成一个。

python单元测试-unittest

Posted on June 9, 2018

python 单元测试 unittest的使用

一。什么是unittest

二。unittest的核心工作模块

test case, test suite, test runner, test fixture。TextTestRunner,HTMLTestRunner(输出html的文件格式)

三。看下面的实例

#!/usr/bin/env python
# -*- coding:utf-8 -*-
import unittest
# unittest.TestCase
# 测试的标识成功是 . 失败是 F ,跳过是s 出错是E
def add(a,b):
    return a+b
def minus(a,b):
    return a-b
def multi(a,b):
    return a*b
def divide(a,b):
    return a/b
class IntegerArimeticTest(unittest.TestCase):
    """docstring for IntegerArimeticTest"""
    def testAdd(self):
        '''test add function'''
        self.assertEqual(3,add(1,2))

    def testminus(self):
        self.assertNotEqual(2,minus(4,1))
    def testmulti(self):
        self.assertEqual(25,multi(5,5))
    def testdivide(self):
        '''test divide function'''
        self.assertEqual(2,divide(5,2))
if __name__ == '__main__':
    unittest.main(verbosity=2)  #  会调用TextTestRunner中的run方法来运行testsuite



四。组织testsuite

下面的代码：
suit=unittest.TestSuite()
tests=[IntegerArimeticTest("testAdd"),IntegerArimeticTest("testminus"),IntegerArimeticTest("testmulti"),IntegerArimeticTest("testdivide")]
suit.addTests(tests)
runner=unittest.TextTestRunner(verbosity=2)
runner.run(suit)

如果将测试用例加到Testsuit中那么执行的顺序就是按照加入的顺序执行的
# 直接用addTest方法添加单个TestCase
suite.addTest(TestMathFunc("test_multi"))
# 用addTests + TestLoader
# loadTestsFromName()，传入'模块名.TestCase名'
suite.addTests(unittest.TestLoader().loadTestsFromName('test_mathfunc.TestMathFunc'))
suite.addTests(unittest.TestLoader().loadTestsFromNames(['test_mathfunc.TestMathFunc']))  # loadTestsFromNames()，类似，传入列表
# loadTestsFromTestCase()，传入TestCase
suite.addTests(unittest.TestLoader().loadTestsFromTestCase(TestMathFunc))
用TestLoader的方法是无法对case进行排序的

五。将结果输出到文件

用例组织好了，但结果只能输出到控制台，这样没有办法查看之前的执行记录，我们想将结果输出到文件。很简单，看示例：

suit=unittest.TestSuite()
tests=[IntegerArimeticTest("testAdd"),IntegerArimeticTest("testminus"),IntegerArimeticTest("testmulti"),IntegerArimeticTest("testdivide")]
suit.addTests(tests)
with open('UnittestTextReport.txt', 'a') as f:
    runner=unittest.TextTestRunner(stream=f,verbosity=2)
    runner.run(suit)

runner=unittest.TextTestRunner(stream=f,verbosity=2)这个是关键。所有程序的执行均输出到此文件，接下来就有测试报告了。

六。test fixture之setUp() tearDown()

上面整个测试基本跑下来了，但可能会有点特殊情况。比如每次执行之前准备环境，或者在每次执行完成后进行清理。总不能每个方法都添加准备环境，清理环境的代码吧。这就涉及到之前的test fixture

def setUp(self):
    print "do something before test.Prepare environment."
def tearDown(self):
    print "do something after test.Clean up."
加入setUp和tearDown表示在执行每个测试用例之前与之后都要执行的操作


如果想要在所有的测试用例之前执行之前检查一次，在执行所有的用例之后执行一次就可使用下面的代码

@classmethod
def setUpClass(cls):
    print "firstly this setUpclass() only called once"
@classmethod
def tearDownClass(cls):
    print "finally:This tearDownClass() method only called once too."

七。跳过某个case

如果我们临时想要跳过某个case,unittest也提供了几个方法
1.skip装饰器

@unittest.skip("i do not want to run this case")
def testdivide(self):
    print "test divide"
    '''test divide function'''
    self.assertEqual(2,divide(5,2))
    self.assertEqual(1,divide(2,2))

将会跳过这个测试用例执行下面的。但是还是可以看到总的skip数量 是 4个但都是divide被跳过了。skip装饰器一共有三个 unittest.skip(reason)、unittest.skipIf(condition, reason)、unittest.skipUnless(condition, reason)，skip无条件跳过，skipIf当condition为True时跳过，skipUn less当condition为False时跳过。

2.除了装饰器，也可以用skipTest()

def testmulti(self):
    self.skipTest("do not run this")
    print "test multi"
    self.assertEqual(25,multi(5,5)) 
    也会跳过该case

八。用HTMLTestRunner输出漂亮的HTML报告

需要外部的库生成html的报告
http://tungwaiyip.info/software/HTMLTestRunner.html
下载后放到python/lib目录下面
或者放到当前文件夹下面

from HTMLTestRunner import *
with open('HtmlReport.html', 'w') as f:
    runner=HTMLTestRunner(stream=f,title='MathFunc Test  Report',description='generated by HTMLTestRunner.',verbosity=2)
    runner.run(suit)
    将测试结果输出到html里面。

python-shutil模块

Posted on April 27, 2018

高级文件操作模块-shutil

背景描述

os模块，已经提供了，对文件的基本的操作，比如对，文件的新建，移动，删除。绝对路径，父目录，文件夹。这些只是最基本的功能，但是并没有提供文件内容复制，压缩，打包，解压。shutil则是对os模块的扩展

shutil的基本方法介绍

1.copy文件内容，到另外一个文件，可以指定拷贝文件的大小。

import shutil
shutil.copyfileobj(fscrc,fdst,length)

其中fscrc,fdst均为文件对象，需要打开文件，才能进行复制，最后的length参数是可选的，指定复制文件内容的大小。

2.copy文件内容，上面的复制很麻烦，还需要自己手动打开文件。open函数打开，接下来看下面的方法。

import shutil
shutil.copyfile(src,dst) src dst 均为文件名

这个dst文件不存在的话，就新建一个文件然后将src的内容写进去。如果存在直接覆盖dst文件的内容

3.copy文件权限，仅仅拷贝src的文件权限到dst

import shutil
shutil.copymod(src,dst) src,dst只能为文件

将文件src的权限直接copy到dst文件，如果dst文件不存在，就会报错。 让dst和src文件拥有相同的权限。

Traceback (most recent call last):
  File "C:\Users\admin\Desktop\resource\baseCode\fileCore\       handler.py", line 7, in <module>
  shutil.copymode('read1.txt','read4.txt')
  File "D:\python2.7\lib\shutil.py", line 91, in copymode
  os.chmod(dst, mode)
  WindowsError: [Error 2] : 'read4.txt'

4.copy文件的所有信息，状态信息。包括权限，组，用户，信息。

import shutil
shutil.copystat(src,dst)  #包括文件的mtime ctime

5.copy 将文件复制到文件夹中

shutil.copy(file,dir) 
#该函数的返回值是复制成功后的字符串格式的文件路径

将file完全移动到dir目录中，只能复制单个文件到指定的路径中。如果后面是文件名，那么dir将会被当成复制后的文件名称。

6.递归复制文件夹中的内容到指定的目录

shutil.copytree(src,dir)
#将src里面的文件夹里的所有的文件及其子目录的内容递归复制到     目标目录。

注意，如果 destination 文件夹已经存在，该操作并返回一个 FileExistsError 错误，提示文件已存在。即表示，如果执行了该函数，程序会自动创建一个新文件夹（destination参数）并将 source 文件夹中的内容复制过去

7.删除文件夹里面的所有内容

shutil.rmtree(path) #删除path目录下的所有东西。

上面的删除要特别注意可以用字符串的endwith()属性来判断。

结论

对于文件和文件夹的操作，最好是os和shutil来结合使用，会达到最佳的效果奥。。。。加油！！！！！
